# コンテキスト効率と Stream タイムアウト対策

このファイルは CLAUDE.md から `@.claude/rules/context-efficiency.md` でインポートされる。
**目的**: `API Error: Stream idle timeout - partial response received` を回避するため、Claude Code が一度に取り込むコンテキスト量と生成トークン量を抑える運用ルールを定める。

## 背景

行数の多いファイルを無批判に全読みするとレスポンスが過大になり、ストリームタイムアウトを誘発する。

## 基本原則

1. **必要な分だけ読む**: 全文理解を目的にせず、タスクに必要な行・関数・定義だけを読む
2. **探索→読解の二段構え**: まず Grep/Glob で該当行を特定し、その行周辺のみ Read する
3. **横断探索は委譲**: 3ファイル以上の横断読みが必要なら、code-explorer サブエージェントに委譲する
4. **委譲時は返却量を明記**: サブエージェント呼び出しのプロンプトに「返す情報の上限」を必ず含める
5. **読解後は要約**: 大きいファイル・複雑なコードを読んだ後は、理解した内容を 2〜3 行で要約してから次のアクションに進む（CLAUDE.md の「中間報告」原則と整合）

## ファイル読解ルール

### Read の使い方

- **500 行を超えるファイル**は全読み禁止。以下のいずれかで局所化する：
  - Grep で該当行番号を特定し、`Read` に `offset` と `limit` を指定して該当周辺（前後 30〜80 行）だけ読む
  - ファイル構造だけ見たい場合は `Grep` で `^(export |function |const |class |interface |type )` を抽出して目次化する
- **1 ターンにつき全読みは 1 ファイルまで**。複数必要なら code-explorer に委譲する
- **画像・バイナリを Read しない**
- 既に把握しているファイルを同ターン内で再読しない。必要なら該当行だけ再読する

### Grep の使い方

- まず `output_mode: "files_with_matches"`（デフォルト）で対象ファイルを絞る
- 内容確認は `output_mode: "content"` + `-n` + `head_limit`（5〜30 程度）を併用し、マッチ量を抑える
- 広すぎるパターン（例: `function`、`const`）はファイル種別 (`type: "ts"` 等) や `glob` でスコープを絞る
- コンテキスト行 `-C` は原則 3〜5 行に留め、10 行以上は避ける

### Glob の使い方

- `src/**/*` のような広いパターンは最初の絞り込み用途に限定する
- 返却件数が 100 を超える場合は、サブディレクトリや拡張子でさらに分割する

### Bash の使い方（探索目的）

- `cat` / `head` / `tail` / `find` / `grep` を探索用途で使わない（専用ツールを使う）
- `wc -l` や `ls -la` で**メタ情報だけ**を取るのは可。ただしファイル全体の中身を標準出力に流す操作は禁止

## サブエージェント委譲ルール

### 委譲すべきケース

- 3ファイル以上の横断読みが必要
- 大きい 1 ファイル（500 行超）の全体像把握が必要
- 複数コンポーネントを跨ぐデータフロー追跡
- 類似機能の洗い出し

### 委譲時のプロンプト必須要件

サブエージェントに渡すプロンプトには、以下を必ず含める：

1. **目的**: 何のためにこの調査をするのか（1 行）
2. **調査スコープ**: 対象ディレクトリ・ファイル・キーワード
3. **返却フォーマット**: 以下のいずれかで上限を明示
   - 「関数シグネチャと 5 行以内の挙動要約のみ」
   - 「該当箇所の `file:line` 参照のみ。コード本文は 10 行以下で必要時のみ引用」
   - 「合計 300 行（または N 箇所）以内で要約」
4. **除外**: 引用不要なもの（例: 「import 文や型定義の列挙は不要」）

### サブエージェントの出力に関する既定値

`code-explorer` / `code-architect` / `code-reviewer` の定義にも出力上限を明記している（各エージェントファイル参照）。呼び出し側で上書きする場合は、上記「委譲時のプロンプト必須要件」のとおりプロンプト内に明示する。

## 生成側（Claude の応答）のルール

- **長大なコード貼付けを避ける**: 差分は最小限で示し、全量貼付けは禁止（Edit ツールは差分だけ送るので優先）
- **既存ファイルの再掲禁止**: 読ませた内容を応答にそのまま貼らない。要約・引用箇所の `file:line` 参照で済ませる
- **中間ログを垂れ流さない**: ツール結果のダンプは避け、要点だけ拾って応答する

## チェックリスト（作業前の自己確認）

大きめのタスクに着手する直前、以下を自問する：

- [ ] 本当に全文読む必要があるか？ Grep で絞れないか？
- [ ] 3ファイル以上跨ぐなら code-explorer に委譲したか？
- [ ] サブエージェントへのプロンプトに「返却量の上限」を書いたか？
- [ ] 同ターン内で既読ファイルを再度全読しようとしていないか？
- [ ] 応答に長大なコード貼付けを含めていないか？
